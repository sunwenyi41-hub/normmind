# NormMind 开发进度追踪

本文档用于管理「规范智能知识问答 Agent」的阶段开发进度。项目采用 `spec-kit` SDD 工作方式推进：每个阶段对应一个独立 spec 单元，开发完成后必须先更新本文档，再进入下一阶段。

## 使用规则

- 每个阶段开始前，先在对应阶段下补充 spec 名称、负责人、开始日期。
- 每个阶段完成后，必须更新本文档中的状态、完成说明、遗留事项和验收结果。
- 若阶段范围发生变化，先更新本文件，再调整对应的 spec、plan、tasks。
- 未在本文件中标记为完成的阶段，不视为已交付。

## 状态说明

- `[ ]` 未开始
- `[~]` 进行中
- `[x]` 已完成
- `[!]` 阻塞中

建议 spec 单元：`foundation-hardening`

目标：把当前已有的登录、问答、Supabase 持久化、Coze 调用链路做稳，形成可持续开发的基础闭环。

状态：`[~] 进行中`

TODO LIST

- [x] 检查并修复历史会话仅显示一条的问题
- [x] 统一会话创建、续问、删除、标题生成逻辑
- [x] 统一登录态恢复、未登录拦截和跳转体验
- [x] 统一 Coze 成功、超时、无引用、异常四类响应处理
- [x] 补充开发用测试数据和最小 seed 数据
- [~] 验证 Supabase RLS、用户数据隔离和 API 所有权校验

验收标准

- 一个账号可以稳定查看多条历史对话
- 新建对话、继续追问、删除对话均可正常使用

完成说明

- 已完成会话创建 / 续问 / 删除 / 标题更新的一体化改造。

- 已统一 Coze 的完成、证据不足、失败与兜底 fallback 行为。
- 已补登录后的安全跳转与 OAuth / 邮箱确认失败提示。
- 已补开发环境一键生成测试会话能力，便于联调与演示。

遗留事项

- 仍需在真实 Supabase 项目上跑一次跨用户越权回归。

## 阶段 2：问答工作台

建议 spec 单元：`qa-workbench`

目标：把当前聊天页升级为专业工作台，突出检索模式、引用追溯、结果处理和多轮追问体验。

状态：`[~] 进行中`

TODO LIST

- [x] 改造首页进入路径与问答工作台主入口
- [x] 完成左侧会话流、筛选区和快速新建对话
- [x] 完成中间问答区，支持普通模式与深度模式切换
- [x] 完成右侧信息栏，展示引用、处理状态和结果操作
- [x] 补充复制、重新生成、反馈、收藏等操作入口
- [x] 补充空状态引导、推荐问题和响应式布局优化
- [ ] 完成阶段验收并更新本文档

验收标准

- 主界面具备专业工作台结构，而非单纯聊天页
- 用户可在同一界面完成提问、查看引用、继续追问和结果处理
- 桌面端与移动端主流程可用

完成说明

- 已切换到 `useChat + DefaultChatTransport + LangChain createAgent().stream()` 主链路。
- 会话主存储已迁移为 Supabase `conversations.messages_json` 快照，历史接口支持从旧 `messages` 行式结构回读兼容。
- Coze 已改为优先走单轮 Bot 调用，旧 Workflow 仅作为兜底 fallback。
- 问答区已升级为工作台结构：左侧会话流，中间问答区，右侧引用 / 处理链路面板。

遗留事项

- 仍需补验证真实 Coze Bot v3 返回结构与当前解析器是否完全一致。
- 仍需补最小 seed 数据、RLS 越权测试与登录态体验回归。

## 阶段 3：引用追溯与 PDF 原文查看

建议 spec 单元：`citation-and-pdf-viewer`

目标：让引用卡片真正能追溯到原文文档，至少支持页码级定位，提升答案可信度。

状态：`[~] 进行中`

TODO LIST

- [x] 统一引用数据结构和前端卡片渲染
- [x] 建立规范文档与本地或存储 PDF 的关联关系
- [x] 实现“查看原文”入口和 PDF 打开流程
- [x] 支持基于页码的原文定位
- [x] 对无法定位页码或段落的情况提供兜底提示
- [ ] 验证移动端和桌面端的 PDF 查看体验
- [ ] 完成阶段验收并更新本文档

验收标准

- 用户点击引用后可以打开关联 PDF
- 至少可定位到具体页码
- 无法精确定位段落时有明确说明，不伪装成精确定位

完成说明

- 已支持引用卡片与本地 PDF 资源映射。
- 已支持基于引用页码或 URL hash 的 PDF 页码定位。
- 已在右侧信息栏中补充原文片段 / PDF 预览双视图。
- 已对“只能定位页码、不能精确到段落”的情况做显式提示，不伪装成精准定位。
- 已补充 PDF 定位提示、可复制的人工核对信息以及页内检索词，降低“打开 PDF 但不知道去哪找”的使用阻力。
- 已优化右侧引用面板在窄屏设备下的布局，移动端会以全宽抽屉展示，PDF 预览高度也已做响应式调整。

遗留事项

- 仍需在真实移动端浏览器中回归 PDF 预览的可用性；当前 Codex 内置浏览器对 localhost 存在连接策略限制，未完成可视化回归截图。
- 仍缺少段落级锚点，高亮能力需要知识库或 Coze 返回更细粒度定位信息。

## 总体决策建议

- 建议内测目标设在阶段 4 完成后进行，这时产品会更像完整的专业工具而非 Demo。
- 若需要尽快试用，可在阶段 3 完成后启动受控内测，但需明确资料库与后台能力尚未完成。
- 默认开发顺序：阶段 1 → 阶段 2 → 阶段 3 → 阶段 4 → 阶段 5 → 阶段 6。

## 进度更新要求

每次完成一个阶段后，必须同步更新以下内容：

- 当前阶段状态：改为 `[x] 已完成`
- 完成说明：记录本阶段交付结果
- 遗留事项：记录进入下一阶段前暂缓的问题
- 若有范围变更：补充调整原因

未更新 `progress.md` 的阶段，视为未完成，不进入下一阶段开发。
